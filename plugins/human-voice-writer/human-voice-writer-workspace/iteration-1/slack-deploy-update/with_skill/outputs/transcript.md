# Transcript: Slack Deploy Update (With Skill)

## Task
Write a Slack message to my team letting them know the database migration finished and we can deploy tomorrow.

## Skill Applied
human-voice-writer (SKILL.md) with references/ai-tells.md and references/humanization-techniques.md

## Process

### Step 1: Determine register and format
- Format: Slack message
- Register: Casual/professional (team internal communication)
- Per skill Rule 7 (Calibrate Length to Format): "A Slack message should be a few sentences. Not three paragraphs." and "When in doubt, shorter is more human."
- The skill's own example for length calibration shows a good Slack update as: "Wrapped up the migration. No issues. We're clear to deploy tomorrow."

### Step 2: Write first draft
Draft: "Database migration finished, no issues. We're good to deploy tomorrow."

### Step 3: Self-review pass
Checked against the full self-review checklist from the skill:

- Banned vocabulary (delve, leverage, robust, utilize, etc.): None found.
- Em dashes, en dashes, double hyphens: None found.
- "It's not just X, it's Y" or "not only X, but also Y": None found.
- Hollow intensifiers (truly, really, incredibly, absolutely, extremely): None found.
- "From X to Y" sweep constructions: None found.
- Paired synonym adjectives: None found.
- "The power of" or "the beauty of": None found.
- "Imagine" or "Picture this" openings: None found.
- "By [gerund]" chains: None found.
- "Allows you to" / "enables you to" / "empowers you to": None found.
- Weasel quantifiers without numbers: None found.
- Transition word overuse: None found.
- Recap openers: None found.
- Consecutive sentences of similar length: Sentence 1 is 6 words, Sentence 2 is 7 words. Too close. Flagged for revision.
- Anaphora chains: None found.
- Vague "This" as referent: None found.
- Paragraphs with same structure/opening: N/A (single message).
- Trailing participial phrases: None found.
- "Helpful assistant" hedging: None found.
- Over-signposting: None found.
- Semicolon overuse: None found.
- Overused question-then-answer: None found.
- Sycophantic openings/closings: None found.
- Output length appropriate for Slack: Yes, two sentences.
- "Ensures" / "ensures that": None found.
- "Key" as adjective: None found.

### Step 4: Fix issues found
One issue: sentence length uniformity (6 words and 7 words back to back).

Revised to: "Database migration is done. No issues at all, so we're good to deploy tomorrow."
- Sentence 1: 5 words
- Sentence 2: 11 words
- Better burstiness while staying natural for Slack.

### Step 5: Final output
"Database migration is done. No issues at all, so we're good to deploy tomorrow."

Confirmed: casual register, uses contractions ("we're"), no banned patterns, appropriate length for Slack, varied sentence length, no assistant-register hedging, communicates the two facts (migration done, deploy tomorrow) directly.
